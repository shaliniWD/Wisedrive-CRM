<analysis>**original_problem_statement:**
The user's initial goal was to build a scalable automotive platform, Wisedrive, which evolved into a monolithic CRM application with an Employee Self-Service (ESS) mobile app.

The scope has since expanded to include:
1.  **New Feature (Leave Management):** Implement a comprehensive leave rules system in the CRM (monthly/quarterly allocation, no carry-forward) and update the ESS mobile app to enforce these rules, display correct balances, and provide user feedback.
2.  **New Feature (Leads Management):** Build a complete 4-phase leads management module.
    *   **Phase 1 (Creation):** Ingest leads from a Twilio WhatsApp webhook, triggered by Meta ads.
    *   **Phase 2 (Assignment):** Assign leads to sales agents via a round-robin system based on city.
    *   **Phase 3 (Follow-up):** Allow agents to update lead status, add notes, and set reminders (with push notifications).
    *   **Phase 4 (Payment):** Integrate Razorpay to generate and send payment links, with a webhook to confirm payment and convert the lead to a customer.
3.  **New Feature (Vehicle Data):** Integrate the Invincible Ocean (Vaahan) API to fetch real-time vehicle registration data within the payment flow.
4.  **New Feature (Dynamic Packages):** Integrate active inspection packages from the settings module into the payment modal, allowing for dynamic selection and price calculation, including discounts.
5.  **Critical Bug Fixes (UAT):** Address multiple high-priority bugs discovered by the user during testing, including a system-wide password reset issue, data synchronization problems, and numerous functional and UI bugs in the CRM's HR Roles module.
6.  **Critical RBAC Fix:** Implement strict role-based access control for Sales Executives, ensuring they can only view leads assigned to them and have a restricted UI (only the Leads tab).

**PRODUCT REQUIREMENTS:**
-   **Leads Module:** Must support 22 distinct lead statuses. Assignment should be round-robin by city. Integrations with Twilio, Razorpay, and Vaahan are mandatory.
-   **RBAC:** Sales Executives must have an own scope for leads, seeing only leads assigned to them. Their UI must be restricted to the leads module, and summary data must reflect their own leads. HR Managers should have a global view.
-   **Payment Modal:** Must fetch real-time vehicle data via the Vaahan API and use dynamic inspection packages for pricing, including discounts.
